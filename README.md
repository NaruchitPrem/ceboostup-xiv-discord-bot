**โครงการพัฒนา Discord Bot ประจำค่าย CE BOOSTUP XIV**
เอกสารฉบับนี้รวบรวมข้อมูลและโครงสร้างของ Discord Bot ที่ใช้สำหรับบริหารจัดการผู้เข้าร่วมค่ายและทีมงาน (Staff) เพื่อส่งต่อเป็นแนวทางให้รุ่นน้องในการพัฒนาค่ายปีถัดไป

**1. Tech Stack & Tools (เครื่องมือที่ใช้ในการพัฒนา)**
- Language: JavaScript (Node.js)
- Library: discord.js (v14)
- Database / Data Source: Google Sheets API (google-spreadsheet, google-auth-library) ดึงข้อมูลจากฟอร์มรับสมัครโดยใช้ Service Account
- Environment Management: dotenv สำหรับซ่อนตัวแปรสำคัญ (Tokens, IDs)
- Hosting / Deployment: Render (ตั้งค่า Web Service)
- Keep-Alive System: สร้าง Dummy HTTP Server (Port 3000) ร่วมกับการใช้ UptimeRobot เพื่อปลุกให้บอททำงาน 24/7

**2. Core Features (ระบบหลักที่พัฒนา)**
2.1 ระบบยืนยันตัวตนอัตโนมัติ (Automated Verification System)
ระบบแยกการทำงานระหว่าง "น้องค่าย" และ "ทีมงาน (Staff)" อย่างชัดเจนเพื่อความปลอดภัยของข้อมูล:
สำหรับน้องค่าย (!setup): ใช้ปุ่มกด (Button) ตรวจสอบผ่าน Discord ID (Username) ที่กรอกไว้ใน Google Sheet หากข้อมูลตรงกัน ระบบจะแจ้งผลการเปิดไพ่ด้วยกล่องข้อความ Embed สุดอลังการ
สำหรับสตาฟ (!setup-staff): ใช้ปุ่มกดร่วมกับฟอร์ม Pop-up (Modal) ตรวจสอบผ่าน รหัสนักศึกษา เพื่อป้องกันคนนอกสวมรอย

2.2 ระบบจัดการยศและชื่อเล่น (Auto Role & Nickname Assignment)
เมื่อยืนยันตัวตนสำเร็จ บอทจะจัดการโปรไฟล์ให้เข้ากับธีมค่ายทันที:

- เปลี่ยนชื่อเล่น (Nickname): เติมคำนำหน้า N' [ชื่อเล่น] สำหรับน้องค่าย และ P' [ชื่อเล่น] สำหรับสตาฟ
- แจกยศสาขา (Major Roles): แบ่งตามหลักสูตร (เช่น วิศวกรรมคอมพิวเตอร์, ไซเบอร์)
- แจกยศบ้าน Arcana (House Roles): จัดสรรน้องๆ และสตาฟเข้าบ้านทั้ง 12 บ้านตามฐานข้อมูล (เช่น The Magician, Justice, The Lovers ฯลฯ)

2.3 ระบบสร้างห้องลับพี่บ้าน
ระบบที่ออกแบบมาเพื่อลดภาระแอดมินในการจัดการห้องแชทของ Staff ประจำบ้าน:
- Event Listener: คอยจับตาดูการเปลี่ยนแปลงยศ (GuildMemberUpdate) ตลอดเวลา
- Logic: เมื่อพบว่าใครมี "ยศ Staff" + "ยศบ้าน" บอทจะทำการแจกยศที่ 3 คือ "ยศพี่บ้านประจำบ้านนั้นๆ" ให้อัตโนมัติ เพื่อใช้ปลดล็อกการมองเห็นห้องแชทลับประจำบ้าน
- Auto-Sync on Start: มีระบบสแกนเช็กย้อนหลังทั้งเซิร์ฟเวอร์ทันทีที่บอทเริ่มทำงาน (ClientReady) เพื่อเก็บตกสตาฟที่ยังไม่ได้ยศพี่บ้าน

2.4 ระบบต้อนรับสมาชิกใหม่ (Welcome Message System)
- ตรวจจับเมื่อมีคนเข้าเซิร์ฟเวอร์ใหม่ (GuildMemberAdd)
- ส่งกล่องข้อความ Embed ต้อนรับสีสันเข้ากับธีมค่าย พร้อมดึงรูป Profile รูปใหญ่ของสมาชิกมาแสดง
- Mention เรียกผู้ใช้เบาๆ พร้อมแนบลิงก์ให้ไปยืนยันตัวตนที่ช่อง Verify ทันที

**3. ข้อควรระวังและการดูแลรักษา (Maintenance Notes)**
- Google Sheet ID & Tabs: หากมีการเปลี่ยนชีทใหม่ในรุ่นถัดไป ต้องแก้ SHEET_ID ในไฟล์ .env และต้องระวังชื่อแท็บด้านล่างของ Excel (เช่น Sheet1) ให้ตรงกับในโค้ดเสมอ
- Discord Role Hierarchy: ยศของตัวบอทในหน้า Server Settings ต้องอยู่ "สูงกว่า" ยศที่บอทจะต้องแจกเสมอ ไม่อย่างนั้นบอทจะ Error: Missing Permissions
- Environment Variables: ห้ามอัปโหลดไฟล์ .env หรือไฟล์ที่มี Private Key ของ Google ขึ้น GitHub เด็ดขาด (ใช้ .gitignore) ให้ไปตั้งค่าในเว็บ Render โดยตรง
